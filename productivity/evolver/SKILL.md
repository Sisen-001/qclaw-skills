# 🧬 Evolver — GEPA风格自我进化技能

> 借鉴：NousResearch/hermes-agent-self-evolution (MIT License)
> 原理：DSPy + GEPA（Genetic-Pareto Prompt Evolution）
> 核心：读取skill → 分析使用轨迹 → 生成候选变体 → 评估筛选 → 自动优化

## 何时触发

用户说：
- "优化这个skill"
- "进化一下"
- "这个skill怎么改进"
- "自我进化"
- "GEPA"

或当你发现某个skill执行效果不好时，主动建议优化。

## 工作原理

```
读取目标Skill ──► 分析最近使用轨迹
                         │
                         ▼
                    GEPA优化器 ◄── 历史对话记录
                         │
                         ▼
                   候选变体生成（3-5个）
                         │
                    约束门检验
                    ① 语义保持（不偏离原目标）
                    ② 规模限制（skill ≤15KB）
                    ③ 无有害内容
                         │
                         ▼
                  最优变体 ──► 备份原文件 ──► 更新
```

## 执行流程

### Step 1 — 选择目标Skill

询问用户要优化哪个skill，或从最近失败的skill自动推断。

### Step 2 — 收集使用轨迹

读取以下来源：
- 该skill最近的执行记录（sessions_history）
- 用户对该skill的修正反馈（corrections.md）
- 该skill的错误日志（如果有）
- 同类skill的对比（如果有）

### Step 3 — 生成候选变体（GEPA突变策略）

对每个skill，生成3个候选变体：

| 突变类型 | 操作 |
|----------|------|
| **精简突变** | 压缩冗余描述，保留核心逻辑（≤15KB） |
| **增强突变** | 补充缺失场景，丰富触发条件 |
| **重构突变** | 重新组织结构，提升可读性 |

### Step 4 — 评估筛选

用以下标准打分（1-5分）：
1. **语义保持**：是否偏离原skill目的？
2. **完整性**：是否覆盖所有原触发场景？
3. **可读性**：结构是否清晰易维护？
4. **简洁性**：是否去除了冗余？
5. **实用性**：执行路径是否最短？

### Step 5 — 更新Skill

- 先备份原文件（`skill.backup.YYYYMMDD.md`）
- 写入最优变体
- 向用户展示改进内容和评分

## 变异提示词模板

```
你是一个AI Agent技能优化专家。请对以下Skill进行GEPA风格的自我进化：

## 原始Skill内容
{原始内容}

## 最近使用轨迹
{使用记录摘要}

## 用户反馈
{反馈内容}

## 任务
生成3个候选变体（精简/增强/重构），每个变体包含：
1. 改动说明
2. 完整Skill文本
3. 自我评分（语义保持/完整性/可读性/简洁性/实用性）

选出最优变体并说明理由。
```

## 约束门规则

每个候选变体必须通过：

| 约束 | 标准 |
|------|------|
| 语义保持 | 核心功能不变 |
| 规模 | SKILL.md ≤ 15KB |
| 安全 | 无有害内容/注入代码 |
| 触发条件 | 保持原有触发关键词 |
| 格式 | 保持标准Skill格式（YAML头/SKILL.md结构） |

## 针对团队Skill的专项进化

### 财姐Skill进化重点

- 优化金股推荐的表述逻辑
- 强化止损/止盈的执行措辞
- 精简数据来源标注格式
- 增加板块逻辑判断的清晰度

### 新妹Skill进化重点

- 优化叙事化写作的示例
- 强化排版规范的精确描述
- 精简公众号发布流程
- 增加互动引导语的吸引力

### 增哥Skill进化重点

- 优化外贸询价的话术
- 强化供应链调研流程
- 精简选品决策树
- 增加Shopify运营的实操细节

## 进化记录格式

每次进化后，记录到 `memory/evolver-log.md`：

```
# 进化日志

## YYYY-MM-DD HH:MM — {Skill名称}

### 原始问题
{为什么需要进化}

### 改进方案
{具体改动}

### 评估结果
| 维度 | 得分 |
|------|------|
| 语义保持 | X/5 |
| 完整性 | X/5 |
| 可读性 | X/5 |
| 简洁性 | X/5 |
| 实用性 | X/5 |
| **总分** | **X/25** |

### 备份文件
{原文件路径}
```

## 注意事项

- 始终先备份再修改
- 语义保持优先于其他优化
- 重大改动须经用户确认后再更新
- 进化后主动测试（如果可执行）
- 记录进化过程到日志
